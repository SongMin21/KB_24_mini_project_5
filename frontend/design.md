# 익명 회고 게시판 — UI 디자인 명세

## 프로젝트 개요

수업이나 프로젝트 종료 후, 익명으로 회고를 작성하고 공유하는 웹 서비스.
회원가입 없이 비밀번호만으로 글/댓글을 관리한다.
카테고리별(배운 것, 부족한 것, 좋았던 것) 글 작성, 공감 버튼, 댓글 기능을 제공한다.

## 디자인 방향

- **톤**: KB 브랜드 아이덴티티를 반영한 신뢰감 있고 따뜻한 분위기.
- **컬러 팔레트** (KB 브랜드 컬러 기준):
  - Primary: #FFBC00 (KB Yellow Positive) — 주요 버튼, 선택된 탭, CTA
  - Primary Hover: #E6A800
  - On Primary(노란 배경 위 텍스트): #2B2313
  - 배운 것(LEARNED): #4CAF50 (그린, 유지)
  - 부족한 것(LACKED): #FF9800 (오렌지, 유지)
  - 좋았던 것(GOOD): #E91E63 (핑크, 유지)
  - Background: #F7F5F0
  - Card: #FFFFFF
  - Text Primary: #545045 (KB Dark Gray)
  - Text Secondary: #60584C (KB Gray)
  - Border: #E6E1D6
  - Danger: #D32F2F
- **폰트**: Pretendard 또는 Noto Sans KR
- **스타일**: 카드 기반 레이아웃, 둥근 모서리(카드 12px / 인풋·일반버튼 8px / 알약형 버튼 999px), 부드러운 그림자
- **헤더**: 흰 배경 유지, "글쓰기" CTA 버튼만 KB Yellow로 강조
- **반응형**: 데스크톱 기준 max-width 960px 중앙 정렬

## 공통 컴포넌트

### Header (모든 페이지 상단)

- 왼쪽: 로고/서비스명 "회고 게시판" (클릭 시 목록 페이지로 이동)
- 오른쪽: "글쓰기" 버튼 (Primary 컬러, 둥근 버튼)
- 하단에 얇은 구분선

### Footer (모든 페이지 하단)

- 중앙 정렬 텍스트: "익명 회고 게시판 ©2026"
- 배경: 투명, 텍스트 색상: #757575
- 상단에 얇은 구분선

**구성 요소**:

1. **페이지 타이틀**: "우리의 회고를 나눠보세요" — 큰 텍스트, 중앙 정렬
2. **필터 탭 바**: 가로 정렬된 4개 탭 버튼
   - 전체: 모든 글 (기본 선택)
   - 날짜별: 선택 시 아래에 date input이 나타남
   - 카테고리별: 선택 시 아래에 카테고리 3개 버튼(배운 것/부족한 것/좋았던 것)이 나타남
   - 공감순: 공감 수 내림차순 정렬
   - 선택된 탭은 Primary 컬러 배경 + 흰색 텍스트, 나머지는 흰색 배경 + 회색 텍스트
3. **날짜 선택 필터** (날짜별 탭 활성 시만 표시): `<input type="date">` + "조회" 버튼
4. **카테고리 필터** (카테고리별 탭 활성 시만 표시): 카테고리별 색상 버튼 3개
5. **게시글 카드 목록**: 세로로 나열
   - 카테고리 뱃지: 각 카테고리 색상의 작은 라벨 (배운 것/부족한 것/좋았던 것)
   - 제목: 볼드, 16px
   - 하단 정보: 공감 수 (하트 아이콘), 댓글 수 (말풍선 아이콘), 작성일
   - 카드 hover 시 약간의 그림자 증가 효과
   - 카드 클릭 시 상세 페이지로 이동
6. **페이지네이션**: 하단 중앙, 숫자 버튼 + 이전/다음 화살표

**구성 요소**:

1. **뒤로가기 링크**: "← 목록으로" — 클릭 시 `/thinking/list`로 이동
2. **게시글 카드** (큰 카드):
   - 카테고리 뱃지: 해당 색상 라벨
   - 제목: 볼드, 20px
   - 본문: 일반 텍스트, 16px, line-height: 1.8
   - 작성일시: 텍스트 Secondary 컬러
   - 하단 액션 버튼:
     - 공감 버튼: 하트 아이콘 + 공감 수, 클릭 시 숫자가 +1 증가 (AJAX, 페이지 새로고침 없음)
     - 수정 버튼: 클릭 시 비밀번호 입력 모달 → 일치하면 수정 페이지로 이동
     - 삭제 버튼: 클릭 시 비밀번호 입력 모달 → 일치하면 삭제 처리
3. **비밀번호 확인 모달** (수정/삭제 클릭 시 팝업):
   - 모달 중앙 배치, 반투명 배경 오버레이
   - "비밀번호를 입력하세요" 텍스트
   - 비밀번호 input (type="password")
   - [확인] [취소] 버튼
4. **댓글 섹션 구분선**: "댓글 N개" 텍스트 + 가로선
5. **댓글 목록**: 카드 형태, 게시글 카드보다 작은 크기
   - 댓글 내용
   - 작성일시
   - 수정/삭제 버튼 (클릭 시 비밀번호 모달)
6. **댓글 수정 모드**: 수정 버튼 클릭 + 비밀번호 확인 후, 댓글 내용이 textarea로 변환되고 [저장] [취소] 버튼 표시
7. **댓글 작성 폼**: 하단 고정형
   - textarea: 댓글 내용 (placeholder: "댓글을 입력하세요")
   - input: 비밀번호 (type="password", placeholder: "비밀번호")
   - 작성 버튼: Primary 컬러

## 인터랙션 상세

### 공감 버튼

- 상세 페이지의 공감 버튼을 클릭하면 페이지 새로고침 없이 숫자만 +1 증가
- 클릭 시 하트 아이콘에 잠깐 커지는 애니메이션 (scale: 1 → 1.3 → 1, 0.3초)
- 버튼 색상이 일시적으로 빨간색으로 변했다가 복귀

### 비밀번호 확인 모달

- 수정/삭제 버튼 클릭 시 모달 팝업
- 모달 외부(오버레이) 클릭 시 모달 닫힘
- 비밀번호 불일치 시 input 아래에 "비밀번호가 일치하지 않습니다" 빨간색 텍스트
- 비밀번호 일치 시:
  - 수정: 수정 페이지로 이동
  - 삭제: "정말 삭제하시겠습니까?" 확인 후 삭제 → 목록으로 이동

### 필터 탭 전환

- 탭 클릭 시 하단 필터 영역이 슬라이드로 나타남
- 날짜별 탭: date input + 조회 버튼
- 카테고리별 탭: 카테고리 색상 버튼 3개 (복수 선택 불가)
- 전체/공감순 탭: 추가 필터 없이 바로 목록 갱신

### 카드 hover

- 목록의 게시글 카드에 마우스 올리면 그림자 증가 + 약간 위로 이동 (translateY: -2px)
- 커서가 pointer로 변경

## 빈 상태 (Empty State)

- 게시글이 없을 때: 일러스트 또는 이모지 + "아직 작성된 회고가 없어요. 첫 번째 회고를 남겨보세요!" + [글쓰기] 버튼
- 댓글이 없을 때: "아직 댓글이 없어요. 첫 번째 댓글을 남겨보세요!" 텍스트
- 검색 결과 없을 때: "해당 조건의 회고가 없어요." 텍스트
